
```markdown

```


### 3. 소개문 수정

**현재:**
```
마이크로서비스 기반 쿠버네티스 모니터링 플랫폼으로, 실시간 관측 인프라(Go Load Balancer, FastAPI 서비스, WebSocket 하트비트)를 AI Agent가 호출 가능한 Tool로 설계한 프로젝트입니다.

**핵심 키워드**: `Kubernetes` | `Go` | `FastAPI` | `MCP Server(설계)` | `LangGraph Agent(설계)`

> AI Agent 자동 장애 진단 설계 → [AI Agent 확장 설계](#ai-agent-확장-설계)
```

**변경:**
소개문은 현재 내용을 기반으로 AI Agent 설계 방향성을 앞에 배치.
키워드 뱃지에서 AI 관련 키워드를 앞으로 이동.

### 4. 섹션 순서 변경

**현재 순서:**
```
1. 프로젝트 구조
2. 서비스 개요
3. 비기능 요구사항
4. 안정성 설계
5. 프록시/집계 최적화
6. 대시보드 하트비트
7. 대시보드 핵심 KPI
8. 트래픽 집계 기준
9. AI Agent 확장 설계  ← 맨 아래
10. 부하 테스트
11. 로컬 실행
12. 의존성
13. 데이터 백업
```

**변경 순서:**
```
1. 프로젝트 구조        (기존 유지)
2. 서비스 개요          (기존 유지)
3. AI Agent 확장 설계   ← 서비스 개요 직후로 이동
4. 비기능 요구사항      (기존 유지)
5. 안정성 설계          (기존 유지)
6. 프록시/집계 최적화    (기존 유지)
7. 대시보드 하트비트     (기존 유지)
8. 대시보드 핵심 KPI    (기존 유지)
9. 트래픽 집계 기준     (기존 유지)
10. 부하 테스트         (기존 유지)
11. 로컬 실행           (기존 유지)
12. 의존성              (기존 유지)
13. 데이터 백업         (기존 유지)
```

이유: 채용담당자가 30초 스캔 시 서비스 구조 파악 후 바로 AI Agent 설계를 볼 수 있도록
